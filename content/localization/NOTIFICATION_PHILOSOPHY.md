# Sabil Notification Philosophy (Phase 5H)

## Purpose

Notifications in Sabil are not for engagement pressure.

They exist to:
- gently reopen a spiritual window
- support sincere return without guilt
- preserve emotional safety after gaps
- keep Quran-centered companionship outside active lessons

## Core Tone

Every notification should feel like:
- warm invitation
- calm emotional pace
- spiritually close language
- non-transactional care

Every notification must avoid:
- urgency
- fear-based motivation
- shame or blame
- productivity framing
- gamified framing

## Guardrails

1. No pressure architecture
- No streak reminders.
- No "you are falling behind" messaging.
- No countdown language.

2. No guilt hooks
- Never frame pauses as failure.
- Never imply Allah's mercy is blocked by inconsistency.

3. Respect user agency
- Reminder settings are optional and user-controlled.
- Time and language are always editable.

4. Preserve beginner safety
- Message should be safe for a spiritually fragile reader.
- Message should allow return after long absence without emotional penalty.

5. Keep message shape short
- One emotional move per line.
- Prefer 8-18 words.
- End with soft landing language.

## Notification Category Model

Use categories to keep copy disciplined:
- opening_window: opens a quiet return moment
- steady_companionship: keeps presence without pressure
- reflective_pause: invites calm breathing and reflection
- gentle_return: used after absence windows

## Absence Return Windows

- 1-3 days: short-pause welcome
- 7-20 days: medium-pause reassurance
- 21+ days: long-pause mercy-centered reopening

All windows must carry the same rule:
- welcome first
- no "catch up" framing
- one gentle next step only

## Multilingual Equivalence

Do not translate literally.

Preserve emotional equivalence between English and Urdu:
- same tenderness
- same calm pacing
- same invitation quality
- same return safety

If either language feels harsher, rewrite both.
