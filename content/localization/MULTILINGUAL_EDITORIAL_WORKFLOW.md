# Sabil Multilingual Editorial Workflow (Phase 5I)

## Purpose

This workflow keeps multilingual scaling emotionally coherent, spiritually safe, and operationally maintainable.

It is intentionally lightweight.

No corporate approval maze.
No productivity pressure language.

## Core Principle

Emotional authenticity outranks literal translation.

If wording is accurate but the heart-feel is wrong, it is not ready.

## Workflow Stages

Each localized language follows this stage path:

1. `untranslated`
- Localized draft does not exist yet.
- English remains canonical source.

2. `draft_localized`
- Localized text exists in metadata and/or blocks.
- Not emotionally reviewed yet.

3. `emotionally_reviewed`
- Emotional review checklist completed.
- Warmth, softness, pacing, and intimacy validated.

4. `qa_approved`
- Localization QA checks completed.
- Cross-language consistency checks completed.

5. `published`
- Publishing safety checks complete.
- Lesson can be published with multilingual confidence.

## Lightweight Authoring Rhythm

1. Draft source day in English with canonical day structure.
2. Add localized Urdu draft for key metadata and blocks.
3. Run emotional review checklist in editor.
4. Run localization QA + cross-language checks.
5. Confirm publishing safety checks.
6. Publish.

## Emotional Review Checklist (Required)

- Warmth stays human and close.
- Softness remains free of guilt/pressure.
- Mobile readability remains calm.
- Emotional equivalence matches source intent.
- Contemplative pacing preserves breathing room.
- Spiritual intimacy remains sincere.

## Cross-Language Consistency Rules

Before QA approval, confirm:
- arc identity aligned
- week identity aligned
- Seerah progression aligned
- emotional pacing aligned

English and Urdu must feel like one journey in two languages.

## Version and Sync Safety

Every lesson carries:
- `content_version`
- `source_revision`
- `source_hash`
- per-language `synced_source_hash`
- `drift_flags`

If localized sync hash differs from source hash after source edits, localization enters drift watch.

## Publishing Safety Rules

Before publish:
- metadata integrity confirmed
- section completeness confirmed
- emotional QA complete
- language consistency complete
- structural alignment complete
- translation sync confirmed

## Operational Notes

- Localization QA dashboard gives visibility, not pressure.
- Drift flags trigger review conversations, not blame.
- Keep checklist discipline strict but calm.
- If uncertain, favor safer and softer language.
