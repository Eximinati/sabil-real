# Future Language Scaling Rules (Phase 5I)

## Purpose

Prepare for Arabic, Turkish, Bahasa, French, and future languages without losing Sabil's emotional identity.

## New Language Onboarding Requirements

Before enabling a new language:
- appoint at least one emotional localization reviewer
- define language-specific warmth/softness guidance
- document forbidden patterns for that language
- create good-vs-bad examples for key surfaces
- define readability standards for mobile

## Required Baseline Artifacts per Language

1. Tone philosophy page
2. Emotional review checklist
3. Anti-pattern list
4. Cross-language consistency notes
5. Localization QA checklist

## Minimum Translation Standards

- No literal-only translation pipeline
- Emotional equivalence with canonical English required
- Reflection prompts must be culturally natural and spiritually gentle
- Return-flow and reminder language must remain non-shaming

## QA Standards

Every new language must pass:
- emotional review checklist
- localization QA checklist
- cross-language consistency checks
- publishing safety checks

## Rollout Guidance

- Start with one pilot week (7 days)
- Run editorial review retrospective
- Fix tone drift before expanding to more days
- Expand in calm batches (not bulk translation pushes)

## Risk Controls

- Use source/sync hash tracking for every localized language
- Track drift flags visibly in localization QA dashboard
- Block publish when safety checks are incomplete
- Keep final authority with human emotional review, not automation
